# TSConfig の解説

## ビルド（トランスパイル）

トランスパイルとは、あるプログラミング言語で書かれたソースコードを、構文が類似した別の言語やバージョンに変換する工程を指します。たとえば、TypeScript では、TypeScript コンパイラ（tsc）を使用して、TypeScript コードを JavaScript に変換します。

また、同じ言語内で異なるバージョン間の変換（例：ES6 → ES5）もトランスパイルと呼ばれます。

デフォルトでは、プロジェクトフォルダ内のすべての `.ts` および `.tsx` ファイルがトランスパイルの対象となり、`node_modules` フォルダ内のファイルは自動的に除外されます。

トランスパイルの対象ファイルを限定するには、`tsconfig.json` のルート項目で `files` または `include` を使用して指定します。
`files` は個々のファイルを明示的に列挙し、`include` はワイルドカード（glob）を使用して、パターンで複数ファイルを指定できます。なお、`files` と `include` の両方が指定されている場合、`files` が優先され、`include` は無視されます。

### files

コンパイル（トランスパイル）する`.ts`ファイルの絶対もしくは相対パスを指定。

以下の例では、`src`フォルダ内`index.ts`と`src/utils`フォルダ内`helper.ts`をコンパイルするように指定しています。

```json
{
  "files": ["./src/index.ts", "./src/utils/helper.ts"]
}
```
